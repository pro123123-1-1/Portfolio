# Task 3: Conduct Sprint Reviews and Retrospectives

## Purpose
To evaluate sprint outcomes, demonstrate features, and refine processes through team reflection.

---

## Sprint Review Process
**Frequency:** End of each sprint  
**Participants:** Full team + stakeholders  
**Duration:** 60 minutes

---

## Sprint 1 Review: Core MVP Features
**Demo Highlights:**
- Product browsing functionality
- Order creation workflow
- Backend APIs (products/orders)
- Full-stack integration

**Feedback Received:**
- Positive: Functionality meets requirements
- Improvement: Enhance UI clarity
- Suggestion: More testing time allocation

---

## Sprint 2 Review: Enhancements
**Demo Highlights:**
- Basic user authentication
- Performance improvements
- Bug fixes
- Updated documentation

**Feedback Received:**
- Positive: System stability improved
- Improvement: Add input validation hints
- Suggestion: Implement loading indicators

---

## Retrospective Methodology
**Format:** Start/Stop/Continue
